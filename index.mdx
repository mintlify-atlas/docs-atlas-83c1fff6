Dokka is the official API documentation engine for Kotlin. It reads KDoc comments in Kotlin source files and Javadoc comments in Java source files, then generates comprehensive API reference documentation in multiple output formats — including its own modern HTML format, Javadoc-compatible HTML, and GitHub Flavored Markdown.

## What is Dokka?

Dokka understands both Kotlin's KDoc syntax and Java's Javadoc comments, making it ideal for mixed-language JVM projects. You can run it via Gradle, Maven, or a standalone CLI tool.

Some projects that use Dokka for their API reference:
- **kotlinx.coroutines** — Kotlin's official concurrency library
- **Ktor** — Kotlin async web framework
- **OkHttp** — Square's HTTP client
- **Gradle** — The build tool itself (Kotlin DSL reference)

## Quick setup

<Steps>
  <Step title="Apply the Gradle plugin">
    Add Dokka to your `build.gradle.kts`:

    ```kotlin build.gradle.kts
    plugins {
        id("org.jetbrains.dokka") version "2.2.0"
    }
    ```
  </Step>
  <Step title="Generate documentation">
    Run the Gradle task to produce HTML documentation:

    ```bash
    ./gradlew :dokkaGenerate
    ```
  </Step>
  <Step title="View the output">
    Open the generated site in your browser. The default output directory is:

    ```
    build/dokka/html/index.html
    ```
  </Step>
</Steps>
