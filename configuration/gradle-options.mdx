Dokka's Gradle DSL lets you tune nearly every aspect of documentation generation — from the project name and output directory down to which visibility modifiers are included and how source links are resolved. Configuration is split across several blocks nested inside the `dokka {}` extension: publication options live in `dokkaPublications {}`, source set options in `dokkaSourceSets {}`, and cross-cutting concerns like source links and per-package rules are nested inside source set blocks.

## General configuration

The top-level `dokka {}` block holds the `dokkaPublications {}` sub-block, which is where you set the module name, output directory, and other publication-wide options. The two built-in publications are `html` and `javadoc`. Both accept the same set of options.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin build.gradle.kts
    plugins {
        id("org.jetbrains.dokka") version "2.2.0"
    }

    dokka {
        dokkaPublications.html {
            moduleName.set(project.name)
            moduleVersion.set(project.version.toString())
            outputDirectory.set(layout.buildDirectory.dir("dokka/html"))
            failOnWarning.set(false)
            suppressInheritedMembers.set(false)
            suppressObviousFunctions.set(true)
            offlineMode.set(false)
            includes.from("packages.md", "extra.md")
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy build.gradle
    plugins {
        id 'org.jetbrains.dokka' version '2.2.0'
    }

    dokka {
        dokkaPublications {
            html {
                moduleName.set(project.name)
                moduleVersion.set(project.version.toString())
                outputDirectory.set(layout.buildDirectory.dir("dokka/html"))
                failOnWarning.set(false)
                suppressInheritedMembers.set(false)
                suppressObviousFunctions.set(true)
                offlineMode.set(false)
                includes.from(files("packages.md", "extra.md"))
            }
        }
    }
    ```
  </Tab>
</Tabs>

<Note>
  When writing Gradle Kotlin DSL in `build.gradle.kts`, type-safe accessors are available (`dokkaPublications.html { ... }`). In a custom Gradle plugin written in a regular `.kt` file, use `dokka.dokkaPublications.named("html") { ... }` instead.
</Note>

### Publication options

<AccordionGroup>
  <Accordion title="moduleName">
    The display name for the project's documentation. It appears in the table of contents, navigation, headers, and log messages. In multi-project builds, each subproject's `moduleName` is used as its section title in the aggregated documentation.

    - **Type:** `String`
    - **Default:** Gradle project name

    ```kotlin
    moduleName.set("my-library")
    ```
  </Accordion>

  <Accordion title="moduleVersion">
    The subproject version displayed in the generated documentation. In single-project builds it is used as the project version. In multi-project builds each subproject's `moduleVersion` is used when aggregating documentation.

    - **Type:** `String`
    - **Default:** Gradle project version

    ```kotlin
    moduleVersion.set(project.version.toString())
    ```
  </Accordion>

  <Accordion title="outputDirectory">
    The directory where the generated documentation is stored. This setting applies to all documentation formats (HTML, Javadoc, etc.) generated by the `dokkaGenerate` task.

    - **Type:** `Directory`
    - **Default:** `build/dokka/html`

    ```kotlin
    // Standard output path
    outputDirectory.set(layout.buildDirectory.dir("dokka/html"))

    // Custom output path using a file tree
    outputDirectory.set(rootDir.resolve("docs/api/0.x"))
    ```

    <Tip>
      For multi-project builds, configure `outputDirectory` in the root project. Use `fileTree` to include multiple Markdown files from a directory at the same time.
    </Tip>
  </Accordion>

  <Accordion title="failOnWarning">
    Whether Dokka should fail the build when a warning is emitted during documentation generation. The process waits until all errors and warnings have been collected before failing.

    - **Type:** `Boolean`
    - **Default:** `false`

    This option works well in combination with `reportUndocumented` in source set configuration.

    ```kotlin
    failOnWarning.set(true)
    ```
  </Accordion>

  <Accordion title="suppressInheritedMembers">
    Whether to suppress inherited members that are not explicitly overridden in a given class. This suppresses functions such as `equals`, `hashCode`, and `toString`, but does **not** suppress synthetic functions such as `dataClass.componentN` and `dataClass.copy`. Use `suppressObviousFunctions` for those.

    - **Type:** `Boolean`
    - **Default:** `false`

    ```kotlin
    suppressInheritedMembers.set(true)
    ```
  </Accordion>

  <Accordion title="suppressObviousFunctions">
    Whether to suppress obvious functions. A function is considered obvious if it is:

    - Inherited from `kotlin.Any`, `Kotlin.Enum`, `java.lang.Object`, or `java.lang.Enum` — such as `equals`, `hashCode`, `toString`.
    - Synthetic (generated by the compiler) and has no documentation — such as `dataClass.componentN` or `dataClass.copy`.

    - **Type:** `Boolean`
    - **Default:** `true`

    ```kotlin
    suppressObviousFunctions.set(true)
    ```
  </Accordion>

  <Accordion title="offlineMode">
    Whether to resolve remote files and links over the network. This includes package-lists used for generating links to external documentation, which allows classes from the standard library to be clickable in your documentation.

    Setting this to `true` can significantly speed up build times in some cases, but will break external class and member links, including links to the Kotlin standard library.

    - **Type:** `Boolean`
    - **Default:** `false`

    <Tip>
      You can cache fetched package-list files locally and supply them to Dokka as local paths via the `externalDocumentationLinks` configuration to get the speed benefit without breaking links.
    </Tip>

    ```kotlin
    offlineMode.set(false)
    ```
  </Accordion>

  <Accordion title="includes">
    A list of Markdown files that contain module-level and package-level documentation. The Markdown files must match the [required format](/configuration/module-and-package-docs). The contents of specified files are parsed and embedded into documentation as module and package descriptions.

    - **Type:** `FileCollection`
    - **Default:** none

    ```kotlin
    // Individual files
    includes.from("packages.md", "extra.md")

    // Using a file tree to include all Markdown files under docs/
    includes.from(
        fileTree("docs") {
            include("**/*.md")
        }
    )
    ```
  </Accordion>
</AccordionGroup>

## Source set configuration

Dokka allows configuring options for each [Kotlin source set](https://kotlinlang.org/docs/multiplatform-discover-project.html#source-sets) independently. Use `dokkaSourceSets.configureEach {}` to apply options to all source sets, or `dokkaSourceSets.named("sourcSetName") {}` to target a specific one.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin build.gradle.kts
    import org.jetbrains.dokka.gradle.engine.parameters.VisibilityModifier

    dokka {
        dokkaSourceSets {
            // Configuration exclusive to the 'linux' source set
            named("linux") {
                dependentSourceSets { named("native") }
                sourceRoots.from(file("linux/src"))
            }

            configureEach {
                suppress.set(false)
                displayName.set(name)
                documentedVisibilities.set(setOf(VisibilityModifier.Public))
                reportUndocumented.set(false)
                skipEmptyPackages.set(true)
                skipDeprecated.set(false)
                suppressGeneratedFiles.set(true)
                jdkVersion.set(8)
                languageVersion.set("1.7")
                apiVersion.set("1.7")
                sourceRoots.from(file("src"))
                classpath.from(file("libs/dependency.jar"))
                samples.from("samples/Basic.kt", "samples/Advanced.kt")
            }
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy build.gradle
    import org.jetbrains.dokka.gradle.engine.parameters.VisibilityModifier

    dokka {
        dokkaSourceSets {
            named("linux") {
                dependentSourceSets { named("native") }
                sourceRoots.from(file("linux/src"))
            }

            configureEach {
                suppress.set(false)
                displayName.set(name)
                documentedVisibilities.set([VisibilityModifier.Public] as Set)
                reportUndocumented.set(false)
                skipEmptyPackages.set(true)
                skipDeprecated.set(false)
                suppressGeneratedFiles.set(true)
                jdkVersion.set(8)
                languageVersion.set("1.7")
                apiVersion.set("1.7")
                sourceRoots.from(file("src"))
                classpath.from(file("libs/dependency.jar"))
                samples.from("samples/Basic.kt", "samples/Advanced.kt")
            }
        }
    }
    ```
  </Tab>
</Tabs>

### Source set options

<AccordionGroup>
  <Accordion title="suppress">
    Whether this source set should be skipped when generating documentation.

    - **Type:** `Boolean`
    - **Default:** `false`
  </Accordion>

  <Accordion title="displayName">
    The display name used to refer to this source set, both in the generated output visible to readers and internally in log messages from `reportUndocumented`.

    - **Type:** `String`
    - **Default:** Deduced from the Kotlin Gradle plugin
  </Accordion>

  <Accordion title="documentedVisibilities">
    Defines which visibility modifiers Dokka includes in the generated documentation. Use this to document `protected`, `internal`, or `private` declarations, or to exclude `public` declarations and only document internal API. You can also use the [`documentedVisibilities()` function](https://github.com/Kotlin/dokka/blob/v2.2.0/dokka-runners/dokka-gradle-plugin/src/main/kotlin/engine/parameters/HasConfigurableVisibilityModifiers.kt) to add visibilities.

    - **Type:** `Set<VisibilityModifier>`
    - **Default:** `VisibilityModifier.Public`
    - **Can be configured per package** via `perPackageOption`

    ```kotlin
    documentedVisibilities.set(
        setOf(VisibilityModifier.Public, VisibilityModifier.Protected)
    )
    // or use the convenience function:
    documentedVisibilities(VisibilityModifier.Public)
    ```
  </Accordion>

  <Accordion title="reportUndocumented">
    Whether to emit warnings about visible undocumented declarations — that is, declarations without KDocs after visibility filters have been applied. Works well with `failOnWarning`. Can also be configured per package.

    - **Type:** `Boolean`
    - **Default:** `false`
  </Accordion>

  <Accordion title="skipEmptyPackages">
    Whether to skip packages that contain no visible declarations after all filters have been applied. For example, if `skipDeprecated` is `true` and a package only contains deprecated declarations, that package is considered empty.

    - **Type:** `Boolean`
    - **Default:** `true`
  </Accordion>

  <Accordion title="skipDeprecated">
    Whether to document declarations annotated with `@Deprecated`. Can also be configured per package.

    - **Type:** `Boolean`
    - **Default:** `false`
  </Accordion>

  <Accordion title="suppressGeneratedFiles">
    Whether to document generated files. Generated files are expected under `{project}/{buildDir}/generated`. When set to `true`, all files in that directory are effectively added to `suppressedFiles`.

    - **Type:** `Boolean`
    - **Default:** `true`
  </Accordion>

  <Accordion title="jdkVersion">
    The JDK version to use when generating external documentation links for Java types. For example, if you use `java.util.UUID` in a public declaration signature and this is set to `8`, Dokka generates a link to the JDK 8 Javadocs for it.

    - **Type:** `Int`
    - **Default:** `8`
  </Accordion>

  <Accordion title="languageVersion">
    The [Kotlin language version](https://kotlinlang.org/docs/compatibility-modes.html) used for setting up analysis and the `@sample` environment.

    - **Type:** `String`
    - **Default:** The latest language version available to Dokka's embedded compiler
  </Accordion>

  <Accordion title="apiVersion">
    The [Kotlin API version](https://kotlinlang.org/docs/compatibility-modes.html) used for setting up analysis and the `@sample` environment.

    - **Type:** `String`
    - **Default:** Deduced from `languageVersion`
  </Accordion>

  <Accordion title="sourceRoots">
    The source code roots to be analyzed and documented. Accepts directories and individual `.kt` and `.java` files.

    - **Type:** `FileCollection`
    - **Default:** Deduced from the Kotlin Gradle plugin

    ```kotlin
    sourceRoots.from(file("src"))
    ```
  </Accordion>

  <Accordion title="classpath">
    The classpath for analysis and interactive samples. Useful when types from dependencies are not resolved or picked up automatically. Accepts both `.jar` and `.klib` files.

    - **Type:** `FileCollection`
    - **Default:** Deduced from the Kotlin Gradle plugin

    ```kotlin
    classpath.from(file("libs/dependency.jar"))
    ```
  </Accordion>

  <Accordion title="samples">
    A list of directories or files containing sample functions referenced via the [`@sample`](https://kotlinlang.org/docs/kotlin-doc.html#sample-identifier) KDoc tag.

    - **Type:** `FileCollection`

    ```kotlin
    samples.from("samples/Basic.kt", "samples/Advanced.kt")
    ```
  </Accordion>
</AccordionGroup>

## Source link configuration

Source links allow readers to navigate from any declaration in the generated documentation directly to the corresponding source line in your remote repository. Configure them inside a `sourceLink {}` block within `dokkaSourceSets.main {}`.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin build.gradle.kts
    dokka {
        dokkaSourceSets.main {
            sourceLink {
                localDirectory.set(file("src/main/kotlin"))
                remoteUrl("https://github.com/your-repo")
                remoteLineSuffix.set("#L")
            }
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy build.gradle
    dokka {
        dokkaSourceSets {
            main {
                sourceLink {
                    localDirectory.set(file("src/main/kotlin"))
                    remoteUrl.set(new URI("https://github.com/your-repo"))
                    remoteLineSuffix.set("#L")
                }
            }
        }
    }
    ```
  </Tab>
</Tabs>

<ParamField path="localDirectory" type="File" required>
  The path to the local source directory. Must be relative to the root of the current project.
</ParamField>

<ParamField path="remoteUrl" type="URI" required>
  The URL of the source code hosting service accessible to documentation readers — such as GitHub, GitLab, or Bitbucket. Used to generate `[source]` links on each declaration.
</ParamField>

<ParamField path="remoteLineSuffix" type="String" default="#L">
  The suffix appended to the URL to point to a specific line number. The line number is appended directly after the suffix. Common values:

  | Service | Suffix |
  |---------|--------|
  | GitHub | `#L` |
  | GitLab | `#L` |
  | Bitbucket | `#lines-` |
</ParamField>

## Per-package options

The `perPackageOption {}` block lets you override source set options for specific packages. Packages are matched using a regular expression supplied to `matchingRegex`.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin build.gradle.kts
    import org.jetbrains.dokka.gradle.engine.parameters.VisibilityModifier

    dokka {
        dokkaPublications.html {
            dokkaSourceSets.configureEach {
                perPackageOption {
                    matchingRegex.set(".*api.*")
                    suppress.set(false)
                    skipDeprecated.set(false)
                    reportUndocumented.set(false)
                    documentedVisibilities.set(setOf(VisibilityModifier.Public))
                }
            }
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy build.gradle
    import org.jetbrains.dokka.gradle.engine.parameters.VisibilityModifier

    dokka {
        dokkaPublications {
            html {
                dokkaSourceSets.configureEach {
                    perPackageOption {
                        matchingRegex.set(".*api.*")
                        suppress.set(false)
                        skipDeprecated.set(false)
                        reportUndocumented.set(false)
                        documentedVisibilities.set([VisibilityModifier.Public] as Set)
                    }
                }
            }
        }
    }
    ```
  </Tab>
</Tabs>

<ParamField path="matchingRegex" type="String" default=".*">
  The regular expression used to match package qualified names. Defaults to `.*`, matching all packages.
</ParamField>

<ParamField path="suppress" type="Boolean" default="false">
  Whether the matched package should be skipped when generating documentation.
</ParamField>

<ParamField path="skipDeprecated" type="Boolean" default="false">
  Whether to document declarations annotated with `@Deprecated` within the matched package.
</ParamField>

<ParamField path="reportUndocumented" type="Boolean" default="false">
  Whether to emit warnings about visible undocumented declarations in the matched package, after visibility filters have been applied. Works with `failOnWarning`.
</ParamField>

<ParamField path="documentedVisibilities" type="Set<VisibilityModifier>" default="VisibilityModifier.Public">
  Visibility modifiers to include in documentation for the matched package. Overrides the source set-level `documentedVisibilities` for this package.
</ParamField>

## External documentation links

The `externalDocumentationLinks {}` block creates clickable links to documentation hosted outside your project. Without it, types from external dependencies appear as unresolved in your documentation. By default, Dokka pre-configures links for the Kotlin standard library, JDK, Android SDK, and AndroidX.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin build.gradle.kts
    dokka {
        dokkaSourceSets.configureEach {
            externalDocumentationLinks.register("example-docs") {
                url("https://example.com/docs/")
                packageListUrl("https://example.com/docs/package-list")
            }
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy build.gradle
    dokka {
        dokkaSourceSets.configureEach {
            externalDocumentationLinks.register("example-docs") {
                url.set(new URI("https://example.com/docs/"))
                packageListUrl.set(new URI("https://example.com/docs/package-list"))
            }
        }
    }
    ```
  </Tab>
</Tabs>

<ParamField path="url" type="URI" required>
  The root URL of the external documentation to link to. **Must** end with a trailing slash. Dokka attempts to resolve the `package-list` automatically from this URL.
</ParamField>

<ParamField path="packageListUrl" type="URI">
  The exact location of a `package-list` file. Use this when automatic resolution fails or when you want to use a locally cached copy to avoid network calls. Package lists contain information about the documented project's subprojects and packages.
</ParamField>

## Complete configuration example

The example below shows every configuration option applied simultaneously, serving as a reference for what a fully-specified Dokka configuration looks like.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin build.gradle.kts
    import org.jetbrains.dokka.gradle.engine.parameters.VisibilityModifier

    plugins {
        id("org.jetbrains.dokka") version "2.2.0"
    }

    dokka {
        dokkaPublications.html {
            moduleName.set(project.name)
            moduleVersion.set(project.version.toString())
            outputDirectory.set(layout.buildDirectory.dir("dokka/html"))
            failOnWarning.set(false)
            suppressInheritedMembers.set(false)
            suppressObviousFunctions.set(true)
            offlineMode.set(false)
            includes.from("packages.md", "extra.md")
        }

        dokkaSourceSets {
            named("linux") {
                dependentSourceSets { named("native") }
                sourceRoots.from(file("linux/src"))
            }

            configureEach {
                suppress.set(false)
                displayName.set(name)
                documentedVisibilities.set(setOf(VisibilityModifier.Public))
                reportUndocumented.set(false)
                skipEmptyPackages.set(true)
                skipDeprecated.set(false)
                suppressGeneratedFiles.set(true)
                jdkVersion.set(8)
                languageVersion.set("1.7")
                apiVersion.set("1.7")
                sourceRoots.from(file("src"))
                classpath.from(file("libs/dependency.jar"))
                samples.from("samples/Basic.kt", "samples/Advanced.kt")

                sourceLink {
                    localDirectory.set(file("src/main/kotlin"))
                    remoteUrl("https://example.com/src")
                    remoteLineSuffix.set("#L")
                }

                externalDocumentationLinks.register("example-docs") {
                    url("https://example.com/docs/")
                    packageListUrl("https://example.com/docs/package-list")
                }

                perPackageOption {
                    matchingRegex.set(".*api.*")
                    suppress.set(false)
                    skipDeprecated.set(false)
                    reportUndocumented.set(false)
                    documentedVisibilities.set(
                        setOf(
                            VisibilityModifier.Public,
                            VisibilityModifier.Private,
                            VisibilityModifier.Protected,
                            VisibilityModifier.Internal,
                            VisibilityModifier.Package
                        )
                    )
                }
            }
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy build.gradle

                suppressInheritedMembers.set(false)

            }
        }

        dokkaSourceSets {
            named("linux") {
                dependentSourceSets { named("native") }
                sourceRoots.from(file("linux/src"))
            }

            configureEach {
                suppress.set(false)
                displayName.set(name)
                documentedVisibilities.set([VisibilityModifier.Public] as Set)
                reportUndocumented.set(false)
                skipEmptyPackages.set(true)
                skipDeprecated.set(false)
    ```
