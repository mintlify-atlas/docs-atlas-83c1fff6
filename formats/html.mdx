HTML is Dokka's default and recommended output format. It produces a fully-featured, searchable API documentation site with support for Kotlin Multiplatform, Android, and Java projects. You can use it for both single-project and multi-project builds, and the output can be hosted on any static web server or service like GitHub Pages.

## Generate HTML documentation

HTML output is supported by all Dokka runners. Choose the approach that matches your build tool.

<Tabs>
  <Tab title="Gradle">
    The Gradle plugin for Dokka includes HTML output by default. Apply the plugin in your `build.gradle.kts`:

    ```kotlin
    plugins {
        id("org.jetbrains.dokka") version "2.2.0"
    }
    ```

    Then run one of the following tasks:

    | Task | Description |
    |------|-------------|
    | `dokkaGenerate` | Generates documentation in all available formats based on applied plugins. Recommended for everyday use — logs a clickable link to the output in IntelliJ IDEA. |
    | `dokkaGeneratePublicationHtml` | Generates HTML only. Exposes the output directory as `@OutputDirectory`, making it composable with other Gradle tasks like packaging a `javadoc.jar` or uploading to a server. |

    ```shell
    ./gradlew :dokkaGenerate
    ```

    <Tip>
      In IntelliJ IDEA you may also see a `dokkaGenerateHtml` task. It is an alias for `dokkaGeneratePublicationHtml` and performs the same operation.
    </Tip>
  </Tab>
  <Tab title="Maven">
    Run the `dokka:dokka` Maven goal to generate documentation in HTML format:

    ```shell
    mvn dokka:dokka
    ```
  </Tab>
  <Tab title="CLI">
    Run the Dokka CLI JAR with the HTML plugin dependencies on the classpath:

    ```bash
    java -jar dokka-cli-2.2.0.jar \
         -pluginsClasspath "./dokka-base-2.2.0.jar;..." \
         ...
    ```
  </Tab>
</Tabs>

<Note>
  HTML pages generated by Dokka must be hosted on a web server to render correctly. You can use any free static site host such as [GitHub Pages](https://docs.github.com/en/pages/getting-started-with-github-pages/about-github-pages). For local preview, use the [built-in IntelliJ IDEA web server](https://www.jetbrains.com/help/idea/php-built-in-web-server.html).
</Note>

## Configure the HTML format

The HTML format is Dokka's base format and provides several configuration options. Set them inside the `pluginsConfiguration.html {}` block.

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin
    // build.gradle.kts
    dokka {
        pluginsConfiguration.html {
            customAssets.from("logo.png")
            customStyleSheets.from("styles.css")
            footerMessage.set("(c) Your Company")
            separateInheritedMembers.set(false)
            templatesDir.set(file("dokka/templates"))
            mergeImplicitExpectActualDeclarations.set(false)
        }
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy
    // build.gradle
    dokka {
        pluginsConfiguration {
            html {
                customAssets.from("logo.png")
                customStyleSheets.from("styles.css")
                footerMessage.set("(c) Your Company")
                separateInheritedMembers.set(false)
                templatesDir.set(file("dokka/templates"))
                mergeImplicitExpectActualDeclarations.set(false)
            }
        }
    }
    ```
  </Tab>
  <Tab title="Maven">
    ```xml
    <plugin>
        <groupId>org.jetbrains.dokka</groupId>
        <artifactId>dokka-maven-plugin</artifactId>
        <configuration>
            <pluginsConfiguration>
                <org.jetbrains.dokka.base.DokkaBase>
                    <customAssets>
                        <asset>${project.basedir}/my-image.png</asset>
                    </customAssets>
                    <customStyleSheets>
                        <stylesheet>${project.basedir}/my-styles.css</stylesheet>
                    </customStyleSheets>
                    <footerMessage>(c) MyOrg 2022 Maven</footerMessage>
                    <separateInheritedMembers>false</separateInheritedMembers>
                    <templatesDir>${project.basedir}/dokka/templates</templatesDir>
                    <mergeImplicitExpectActualDeclarations>false</mergeImplicitExpectActualDeclarations>
                </org.jetbrains.dokka.base.DokkaBase>
            </pluginsConfiguration>
        </configuration>
    </plugin>
    ```
  </Tab>
  <Tab title="CLI">
    Via command line options:

    ```bash
    java -jar dokka-cli-2.2.0.jar \
         -pluginsConfiguration "org.jetbrains.dokka.base.DokkaBase={\"customAssets\": [\"my-image.png\"], \"customStyleSheets\": [\"my-styles.css\"], \"footerMessage\": \"(c) 2022 MyOrg\", \"separateInheritedMembers\": false, \"templatesDir\": \"dokka/templates\", \"mergeImplicitExpectActualDeclarations\": false}"
    ```

    Or via JSON configuration:

    ```json
    {
      "moduleName": "Dokka Example",
      "pluginsConfiguration": [
        {
          "fqPluginName": "org.jetbrains.dokka.base.DokkaBase",
          "serializationFormat": "JSON",
          "values": "{\"customAssets\": [\"my-image.png\"], \"customStyleSheets\": [\"my-styles.css\"], \"footerMessage\": \"(c) 2022 MyOrg\", \"separateInheritedMembers\": false, \"templatesDir\": \"dokka/templates\", \"mergeImplicitExpectActualDeclarations\": false}"
        }
      ]
    }
    ```
  </Tab>
</Tabs>

### Configuration options

| Option | Description |
|--------|-------------|
| `customAssets` | Paths to image assets bundled with the documentation. Files are copied to the `<output>/images` directory. Accepts any file extension. |
| `customStyleSheets` | Paths to `.css` stylesheets bundled with the documentation and applied to every page. |
| `templatesDir` | Path to a directory containing custom FreeMarker HTML templates that override Dokka's defaults. |
| `footerMessage` | Text displayed in the footer of every page. |
| `separateInheritedMembers` | When `true`, inherited properties and functions are listed in a separate section from directly declared members. Defaults to `false`. |
| `mergeImplicitExpectActualDeclarations` | When `true`, declarations that share a fully qualified name but are not marked as `expect`/`actual` are merged into a single entry. Useful for legacy codebases. Defaults to `false`. |

## Customize the HTML output

The HTML format supports several customization points for branding and layout.

### Styles

Provide your own CSS files using the `customStyleSheets` option. They are applied globally to every page. You can also replace Dokka's built-in stylesheets by supplying a file with one of the reserved names:

| Stylesheet | Description |
|------------|-------------|
| `style.css` | Main stylesheet covering the majority of page styles |
| `logo-styles.css` | Header logo styling |
| `prism.css` | Styles for the PrismJS syntax highlighter |

### Assets

Use `customAssets` to bundle image files alongside the generated docs:

```kotlin
customAssets.from("example.png", "example2.png")
```

To replace Dokka's default logo, provide a file named `logo-icon.svg`. The maximum supported dimensions are 120×36 px — larger images are resized automatically.

### Logo

<Steps>
  <Step title="Provide a custom logo asset">
    Add your logo file (`.svg` or `.png`) to the `customAssets` list and name it `logo-icon.svg` to replace the default:

    ```kotlin
    customAssets.from("my-logo.svg")
    ```
  </Step>
  <Step title="Override logo styles (optional)">
    If you need to adjust sizing, padding, or switch from `.svg` to `.png`, override `logo-styles.css` via `customStyleSheets`:

    ```kotlin
    customStyleSheets.from("logo-styles.css")
    ```
  </Step>
</Steps>

### Footer

Set the footer text using the `footerMessage` option:

```kotlin
footerMessage.set("(c) 2024 Acme Corp")
```

### FreeMarker templates

Dokka renders HTML pages using [FreeMarker](https://freemarker.apache.org/) templates. You can override any template by placing a file with the same name in the directory pointed to by `templatesDir`. Dokka falls back to its built-in templates for any file not found in that directory.

| Template | Description |
|----------|-------------|
| `base.ftl` | Root layout applied to all pages |
| `includes/header.ftl` | Page header — logo, version selector, theme toggle, and search |
| `includes/footer.ftl` | Page footer — `footerMessage` and copyright text |
| `includes/page_metadata.ftl` | Metadata injected into the `<head>` element |
| `includes/source_set_selector.ftl` | Source set selector shown in the header for Kotlin Multiplatform docs |

Templates have access to the following variables:

| Variable | Description |
|----------|-------------|
| `${pageName}` | The title of the current page |
| `${footerMessage}` | The value of the `footerMessage` configuration option |
| `${sourceSets}` | Nullable list of source sets for multi-platform pages. Each item has `name`, `platform`, and `filter` properties. |
| `${projectName}` | Project name — available only inside a `template_cmd` directive |
| `${pathToRoot}` | Relative path from the current page to the output root — available only inside a `template_cmd` directive |

Use the `template_cmd` directive for variables that require additional context at render time:

```html
<@template_cmd name="projectName">
    <span>${projectName}</span>
</@template_cmd>
```

Dokka also provides the following custom directives:

| Directive | Description |
|-----------|-------------|
| `<@content/>` | Renders the main page content |
| `<@resources/>` | Injects scripts and stylesheets |
| `<@version/>` | Renders the version number, or a version navigator when the versioning plugin is applied |
