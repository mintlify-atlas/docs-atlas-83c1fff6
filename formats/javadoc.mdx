Dokka's Javadoc output format produces HTML pages that closely resemble the classic Javadoc HTML format generated by the standard Java `javadoc` tool. It is not a pixel-perfect copy, but it is visually close enough that consumers familiar with Javadoc-style docs will feel at home immediately. All Kotlin code and signatures are translated to their Java equivalents via the bundled Kotlin-as-Java plugin, so the rendered pages reflect how the API appears from the Java side.

<Warning>
  The Javadoc output format is in **Alpha** stability. You may encounter bugs and migration issues. Successful integration with tools that consume Javadoc as input (such as IDE indexers or documentation hosting services) is not guaranteed. Use it at your own risk.
</Warning>

<Note>
  The Javadoc format does not support multi-project builds or Kotlin Multiplatform projects. Use the [HTML format](/formats/html) for those scenarios.
</Note>

## How it works

The format is implemented as a Dokka plugin, maintained by the Dokka team, and open-sourced on [GitHub](https://github.com/Kotlin/dokka/tree/master/dokka-subprojects/plugin-javadoc). Under the hood it relies on the [Kotlin-as-Java plugin](https://github.com/Kotlin/dokka/tree/master/dokka-subprojects/plugin-kotlin-as-java), which is bundled and applied automatically when you enable the Javadoc format. This means every Kotlin-specific construct — extension functions, top-level functions, data classes, and so on — is presented with its corresponding Java representation on the rendered page.

## Enable and generate Javadoc documentation

<Tabs>
  <Tab title="Gradle">
    The Gradle plugin for Dokka ships with Javadoc support included. Add the `org.jetbrains.dokka-javadoc` plugin ID alongside (or instead of) the standard `org.jetbrains.dokka` plugin:

    ```kotlin
    // build.gradle.kts
    plugins {
        kotlin("jvm") version "2.3.0"
        id("org.jetbrains.dokka-javadoc") version "2.2.0"
    }
    ```

    Once the plugin is applied, you can run either of the following tasks:

    | Task | Description |
    |------|-------------|
    | `dokkaGenerate` | Generates documentation in all formats based on the applied plugins |
    | `dokkaGeneratePublicationJavadoc` | Generates Javadoc output only |

    ```shell
    ./gradlew :dokkaGeneratePublicationJavadoc
    ```

    A full working example is available in the [Dokka repository](https://github.com/Kotlin/dokka/tree/master/examples/gradle-v2/javadoc-example). Its `build.gradle.kts` shows a typical setup including a module description file and source links:

    ```kotlin
    plugins {
        kotlin("jvm") version "2.3.0"
        id("org.jetbrains.dokka-javadoc") version "2.2.0"
    }

    dokka {
        // Used as the project name in the header
        moduleName.set("Dokka Gradle Javadoc Example")

        dokkaSourceSets.main {
            // Module and package descriptions
            includes.from("Module.md")

            // Source links pointing back to the repository
            sourceLink {
                localDirectory.set(file("src/main/kotlin"))
                remoteUrl("https://github.com/Kotlin/dokka/tree/master/examples/gradle/dokka-gradle-javadoc-example/src/main/kotlin")
                remoteLineSuffix.set("#L")
            }
        }
    }
    ```
  </Tab>
  <Tab title="Maven">
    The Maven plugin for Dokka includes Javadoc support out of the box. No additional dependencies are needed. Use the following goals:

    | Goal | Description |
    |------|-------------|
    | `dokka:javadoc` | Generates documentation in Javadoc format |
    | `dokka:javadocJar` | Generates a `javadoc.jar` file containing the Javadoc documentation |

    ```shell
    mvn dokka:javadoc
    ```

    To produce a `javadoc.jar` ready for publishing to Maven Central:

    ```shell
    mvn dokka:javadocJar
    ```
  </Tab>
  <Tab title="CLI">
    Because the Javadoc format is a Dokka plugin, you must download its JAR and pass it to `pluginsClasspath`. The Javadoc plugin also requires two additional JARs:

    - [`kotlin-as-java-plugin`](https://repo1.maven.org/maven2/org/jetbrains/dokka/kotlin-as-java-plugin/)
    - [`korte-jvm`](https://repo1.maven.org/maven2/com/soywiz/korlibs/korte/korte-jvm/3.3.0/korte-jvm-3.3.0.jar)

    Via command line options:

    ```bash
    java -jar dokka-cli-2.2.0.jar \
         -pluginsClasspath "./dokka-base-2.2.0.jar;...;./kotlin-as-java-plugin-2.2.0.jar;./korte-jvm-3.3.0.jar;./javadoc-plugin-2.2.0.jar" \
         ...
    ```

    Via JSON configuration:

    ```json
    {
      "pluginsClasspath": [
        "./dokka-base-2.2.0.jar",
        "...",
        "./kotlin-as-java-plugin-2.2.0.jar",
        "./korte-jvm-3.3.0.jar",
        "./javadoc-plugin-2.2.0.jar"
      ]
    }
    ```
  </Tab>
</Tabs>

## Build a javadoc.jar for publishing

Many artifact repositories, including Maven Central, require a `javadoc.jar` alongside the library JAR. You can produce one using a custom Gradle task that consumes the `dokkaGeneratePublicationJavadoc` output directory:

<Tabs>
  <Tab title="Kotlin DSL">
    ```kotlin
    // build.gradle.kts
    val dokkaJavadocJar by tasks.registering(Jar::class) {
        description = "A Javadoc JAR containing Dokka Javadoc"
        from(tasks.dokkaGeneratePublicationJavadoc.flatMap { it.outputDirectory })
        archiveClassifier.set("javadoc")
    }
    ```
  </Tab>
  <Tab title="Groovy DSL">
    ```groovy
    // build.gradle
    tasks.register('dokkaJavadocJar', Jar) {
        description = 'A Javadoc JAR containing Dokka Javadoc'
        from(tasks.named('dokkaGeneratePublicationJavadoc').flatMap { it.outputDirectory })
        archiveClassifier.set('javadoc')
    }
    ```
  </Tab>
</Tabs>

<Tip>
  If you publish to Maven Central, [javadoc.io](https://javadoc.io/) can host your library's API docs for free directly from the `javadoc.jar`, with no additional configuration required.
</Tip>
